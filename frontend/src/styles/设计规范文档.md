# 设计规范文档

## 1. 设计理念

### 1.1 核心原则
- **简洁美观**：界面简洁明了，避免不必要的视觉元素，突出核心功能
- **一致性**：所有模块保持统一的设计语言和交互模式
- **响应式**：适配不同屏幕尺寸，确保在各种设备上的良好体验
- **可访问性**：确保界面对所有用户都友好，包括视觉障碍用户
- **性能优先**：设计考虑性能影响，避免过度复杂的动画和效果

### 1.2 目标用户
- 平台管理员和运营人员
- 技术背景的用户，对管理界面有较高的效率要求
- 需要快速获取信息和执行操作的用户

## 2. 色彩系统

### 2.1 主色调
- **主色**：#3b82f6（蓝色）
  - 用途：主要按钮、关键操作、高亮状态
  - 渐变：135度渐变，从#3b82f6到#2563eb
  - 深色模式：保持不变

### 2.2 辅助色
- **成功**：#10b981（绿色）
  - 用途：成功状态、积极反馈、在线状态
  - 渐变：135度渐变，从#10b981到#059669

- **警告**：#f59e0b（橙色）
  - 用途：警告状态、提醒信息
  - 渐变：135度渐变，从#f59e0b到#d97706

- **危险**：#ef4444（红色）
  - 用途：错误状态、删除操作、负面反馈
  - 渐变：135度渐变，从#ef4444到#dc2626

- **信息**：#6366f1（紫色）
  - 用途：信息提示、次要操作
  - 渐变：135度渐变，从#6366f1到#4f46e5

### 2.3 中性色
- **背景色**：#0f172a（深蓝灰）
  - 用途：页面主背景

- **卡片背景**：#1e293b（中蓝灰）
  - 用途：卡片、面板背景

- **元素背景**：#334155（浅蓝灰）
  - 用途：输入框、下拉菜单、按钮背景

- **边框色**：rgba(255, 255, 255, 0.08)
  - 用途：卡片边框、分割线

- **文本主色**：#f8fafc（白色）
  - 用途：主要标题、正文文本

- **文本次要色**：#94a3b8（浅灰）
  - 用途：副标题、辅助文本、提示信息

- **文本禁用色**：#64748b（深灰）
  - 用途：禁用状态、占位符文本

## 3. 字体系统

### 3.1 字体选择
- **主要字体**：系统默认无衬线字体
  - 中文：微软雅黑、苹方、思源黑体
  - 英文：San Francisco、Roboto、Arial

### 3.2 字体层级

| 级别 | 字体大小 | 字重 | 行高 | 用途 | 示例 |
|------|---------|------|------|------|------|
| H1 | 24px | 700 | 32px | 页面大标题 | 数据概览 |
| H2 | 20px | 600 | 28px | 卡片标题、区域标题 | 用户管理 |
| H3 | 18px | 600 | 24px | 子标题、表格标题 | 快捷功能 |
| H4 | 16px | 600 | 20px | 小卡片标题 | 服务名称 |
| 正文 | 14px | 400 | 20px | 普通文本、表格内容 | 用户名、描述 |
| 小字 | 12px | 500 | 16px | 辅助信息、标签 | 较上月、状态标签 |

## 4. 间距系统

### 4.1 基础间距
- **4px**：最小间距单位，用于小图标、小元素间距
- **8px**：小间距，用于按钮内边距、图标与文本间距
- **12px**：中小组件间距，用于表单元素、卡片内边距
- **16px**：标准间距，用于卡片间距、区域间距
- **20px**：大间距，用于主要区域间距、标题与内容间距
- **24px**：超大间距，用于页面边缘间距、主要内容区块间距

### 4.2 布局间距
- **页面边距**：20px（大屏幕）、16px（中等屏幕）、12px（小屏幕）
- **卡片间距**：16-24px
- **组件间距**：8-16px
- **行间距**：基于字体大小的1.4-1.5倍

## 6. 交互设计

### 6.1 动画效果

- **过渡动画**：
  - 持续时间：0.3s
  - 缓动函数：cubic-bezier(0.4, 0, 0.2, 1)
  - 应用场景：卡片hover、按钮状态变化、模态框显示/隐藏

- **加载动画**：
  - 颜色：#3b82f6
  - 样式：旋转圆环
  - 背景：rgba(15, 23, 42, 0.8)，带模糊效果

- **脉冲动画**：
  - 持续时间：2s
  - 应用场景：徽章提示、在线状态指示器

- **渐变动画**：
  - 持续时间：0.5s
  - 应用场景：按钮hover时的光效

### 6.2 反馈机制

- **成功反馈**：
  - 颜色：#10b981
  - 图标：el-icon-success
  - 背景：rgba(16, 185, 129, 0.1)
  - 边框：rgba(16, 185, 129, 0.2)

- **错误反馈**：
  - 颜色：#ef4444
  - 图标：el-icon-error
  - 背景：rgba(239, 68, 68, 0.1)
  - 边框：rgba(239, 68, 68, 0.2)

- **警告反馈**：
  - 颜色：#f59e0b
  - 图标：el-icon-warning
  - 背景：rgba(245, 158, 11, 0.1)
  - 边框：rgba(245, 158, 11, 0.2)

- **信息反馈**：
  - 颜色：#3b82f6
  - 图标：el-icon-info
  - 背景：rgba(59, 130, 246, 0.1)
  - 边框：rgba(59, 130, 246, 0.2)

### 6.3 状态管理

- **加载状态**：
  - 全局加载：全屏遮罩，居中加载动画
  - 局部加载：表格、表单等区域的加载动画

- **空状态**：
  - 图标：对应功能的相关图标，64px大小
  - 文字："暂无数据"或类似提示，16px
  - 背景：#334155，带圆角和边框

- **错误状态**：
  - 图标：el-icon-warning-outline，48px大小
  - 标题："加载失败"，20px
  - 描述：具体错误信息，14px
  - 操作：重试按钮

## 7. 响应式设计

### 7.1 断点设置

| 断点 | 屏幕宽度 | 布局调整 |
|------|---------|----------|
| 大屏幕 | ≥1200px | 完整布局，多列网格 |
| 中等屏幕 | 1024px-1199px | 适当调整间距，减少列数 |
| 小屏幕 | 768px-1023px | 单列布局，垂直排列元素 |
| 移动设备 | <768px | 紧凑布局，简化界面元素 |

### 7.2 布局策略

- **网格系统**：
  - 使用CSS Grid和Flexbox结合
  - 大屏幕：多列布局，充分利用空间
  - 小屏幕：单列布局，垂直排列

- **组件适配**：
  - 按钮：小屏幕上使用图标按钮，节省空间
  - 表单：小屏幕上输入框宽度100%
  - 表格：小屏幕上简化列数，或使用横向滚动
  - 卡片：小屏幕上垂直堆叠，减少内边距

- **字体适配**：
  - 大屏幕：使用完整字体层级
  - 小屏幕：适当减小字体大小，保持可读性

- **间距适配**：
  - 大屏幕：宽松间距，提升视觉舒适度
  - 小屏幕：紧凑间距，最大化内容展示

## 9. 代码规范

### 9.1 CSS规范

- **命名约定**：使用BEM命名规范
  - 示例：.page-card__header

- **缩进**：4空格缩进
- **分号**：每条CSS规则结束添加分号
- **引号**：使用双引号
- **大小写**：选择器小写，属性名小写
- **注释**：关键样式添加注释说明

### 9.2 Vue组件规范

- **文件名**：PascalCase命名
  - 示例：UserList.vue

- **组件名**：PascalCase命名，与文件名一致
  - 示例：AdminUserList

- **Props**：camelCase命名，添加类型和默认值
- **Events**：kebab-case命名
- **模板**：使用4空格缩进，合理使用v-if/v-for
- **脚本**：使用ES6语法，合理组织方法和数据

### 9.3 响应式代码示例

```css
/* 大屏幕布局 */
@media (min-width: 1200px) {
  .stats-grid {
    grid-template-columns: repeat(4, 1fr);
    gap: 24px;
  }
}

/* 中等屏幕布局 */
@media (max-width: 1024px) {
  .stats-grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 16px;
  }
  
  .admin-page {
    padding: 16px;
  }
}

/* 小屏幕布局 */
@media (max-width: 768px) {
  .stats-grid {
    grid-template-columns: 1fr;
    gap: 12px;
  }
  
  .admin-page {
    padding: 12px;
  }
  
  .page-card {
    padding: 16px;
  }
}
```

## 11. 设计检查清单

### 11.1 视觉一致性检查
- [ ] 所有模块使用统一的色彩系统
- [ ] 所有组件遵循统一的尺寸和间距规范
- [ ] 字体层级在整个应用中保持一致
- [ ] 图标使用符合规范，尺寸统一
- [ ] 卡片、按钮等组件样式统一

### 11.2 交互一致性检查
- [ ] 所有按钮具有相同的hover和点击效果
- [ ] 表单元素具有统一的focus状态
- [ ] 加载状态和错误提示样式一致
- [ ] 表格和列表的交互模式统一
- [ ] 动画效果流畅且一致

### 11.3 响应式检查
- [ ] 页面在大屏幕上布局合理
- [ ] 页面在中等屏幕上适配良好
- [ ] 页面在小屏幕上可正常使用
- [ ] 组件在不同尺寸下保持可用性
- [ ] 字体和间距在不同设备上适当调整

### 11.4 可访问性检查
- [ ] 文本与背景对比度符合WCAG标准
- [ ] 所有交互元素可通过键盘访问
- [ ] 图标有适当的文本描述
- [ ] 颜色不是唯一的信息传达方式
- [ ] 加载状态有明确的视觉反馈
